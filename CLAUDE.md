Full plan: `/Users/apple/.claude/plans/iterative-swimming-treasure.md`

## Coding Conventions
- PascalCase for class names (`Student`, `Course`, `StudentManager`)
- Private attributes use `__` prefix (e.g. `self.__name`)
- Always add comments explaining *why* a concept is being used — this is a learning project
- Keep the interactive menu working and runnable after every phase

## User Context
- Learning Python OOP and core concepts through a real project
- Prefers building piece by piece (one concept per session), not all at once
- After each phase: build → run → break intentionally → explain → try a variation
